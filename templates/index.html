Complete CI/CD Pipeline Dashboard

This CI/CD pipeline automates the deployment of a Flask application on Minikube. Triggered by a push to the main branch, GitHub Actions first checks out the code and extracts a shortened commit SHA for versioning. It then logs into Docker Hub, builds a Docker image tagged with the commit SHA, and pushes it to the repository. The pipeline uses yq to update the Kubernetes deployment manifest with the new image tag, and commits the updated manifest back to Git. Finally, Argo CD syncs the repository with the Minikube cluster, deploying the updated application to Kubernetes, ensuring it is running live. Each stage is tracked with clear success messages, providing visibility into the full CI/CD lifecycle.
